# Code of Conduct

KORA is an early alpha open-source project. The project works best when contributors keep discussion respectful, practical, and focused on improving the software and documentation.

## Expected behavior

- Be respectful and constructive.
- Assume good intent, but ask for clarification when scope or intent is unclear.
- Keep feedback specific and actionable.
- Respect maintainer decisions about project scope, especially around public surface expansion.
- Help keep discussions focused on KORA's deterministic-first execution model and current alpha goals.

## Unacceptable behavior

The following behavior is not acceptable:

- Harassment, threats, or abusive language
- Personal attacks or repeated hostile comments
- Spam, trolling, or off-topic disruption
- Bad-faith contributions intended to waste maintainer time
- Publishing private information without permission

## Reporting concerns

If you see behavior that violates this code of conduct, contact the maintainers directly through the repository owner.

For now, KORA does not have a separate community moderation process. Maintainers may remove comments, close issues, or block participation when needed to keep the project usable and respectful.

## Scope

This code of conduct applies to project spaces such as issues, pull requests, discussions, and other KORA-maintained collaboration channels.
