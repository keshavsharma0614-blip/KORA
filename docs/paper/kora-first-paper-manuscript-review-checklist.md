# KORA First Paper Manuscript Review Checklist

## Claim Safety

- Approved claim only: KORA reduced model invocations by 80% in a reproducible deterministic-heavy benchmark workload.
- Local/no-network validation wording is used for synthetic validation examples.
- Non-claims are preserved.
- No production/API-cost/energy proof claims are added.
- No broad workload superiority claim is added.
- No real provider validation claim is made from local/no-network examples.

## Evidence Consistency

- Deterministic-heavy benchmark values match the evidence summary.
- Customer-support values match workload docs and local validation reviewer packet.
- Local/no-network label is used correctly.
- KORA Studio is described only as future visualization or workflow context, not evidence.
- Latency language remains directional until measured p50/p95 data exists.

## Paper Quality

- Thesis is clear: deterministic-first execution control before inference.
- Metrics are defined before results are interpreted.
- Limitations are visible and specific.
- Future work is honest about missing evidence.
- Reviewer concerns about routing, caching, provider validation, and cost claims are answered without overclaiming.

## Before arXiv/workshop submission

- Complete references.
- Add figures.
- Add repeated runs if available.
- Add latency table if measured.
- Confirm license/citation info.
- Final internal claim review.
